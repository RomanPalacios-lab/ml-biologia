Introducción al uso de machine learning en biología

Capitulo 2 Capítulo 2: ¿Qué es Machine Learning?

Como se ha mencionado anteriormente el enfoque de este libro es aprender a usar las herramientas que machine learning provee para resolver problemas o responder preguntar en biología. Sin embargo, aún no hemos dado una definición de los que es machine learning. En pocas palabras, machine learning se refiere al uso de analisis estadísticos para hacer predicciones o inferencias sobre un problema en particular utilizando utilizando particiones del conjunto de datos.

Actualmente, las aplicaciones de machine learning se encuentran varios ámbitos demuestra vida diaria. Cuando hablamos a nuestro celular para interactuar con Alexa (Amazon), Siri (Apple) o Google Maps, el machine learning utiliza esas palabras para crear matrices de 1s y 0s para que sean interpretados por el celular. Igualmente, Google Maps colecta datos de tráfico desde el punto en el que estamos al sitio en donde vamos para encontrar la ruta más rápido al lugar de destino. De ese mismo modo, hay cientos de ejemplos del uso de machine learning en cosas que usamos en nuestro día a día.

No obstante, al momento de utilizar modelos para machine learning debemos tener en cuenta que hay costos de oportunidades (conocidos como tradeoffs en inglés). Este es el caso para el trio de flexibilidad-interpretabilidad-complejidad para la estimación de la función de interés. Flexibilidad hace referencia a cuanto las características de los datos influencia al modelo, dado esto, algunos modelos lineales son más flexibles que otros; modelos más flexibles pueden generar más formas para estimar nuestra función de interés. La interpretabilidad por otro lado se refiere a lo fácil o difícil que es entender como las variables influencian el resultado. De esto modo, podemos decir que modelos mas inflexibles son mas interpretables; la flexibilidad de un modelo aumenta a medida que la interpretabilidad disminuye. Por último, la complejidad hacer referencia al número de variables que se usan en el modelo para realizar predicciones certeras; modelos lineales simples son menos complejos que modelos lineales con más de dos variables.

Igualmente, otros dos conceptos para tener en cuenta cuando se usan modelos de machine learning son el sesgo y la varianza. Sesgo se refiere al error que se da a causa por aproximar un problema de la vida real, dado que los problemas de la vida real no son necesariamente explicados por una relación lineal simple, lo cual resulta en un sesgo en la estimación de la función. Por otro lado, la varianza hace referencia a la cantidad que una función cambiaria si se estimara con set de datos de entrenamiento diferente. Dado que diferente set de datos de entrenamiento causaría resultados diferentes en la variable repuesta de la función. Lo que buscamos al elegir el método de aprendizaje estadístico de interés es aquel que tenga la menor varianza y el menor sesgo. Otro concepto para tomar en cuenta cuando se esta eligiendo el método de aprendizaje estadístico es la maldición de la dimensionalidad, el cual se refiere a los problemas que se dan a causa del aumento de variables independientes cuando en un set de datos. Debido a que al aumentar el número de dimensionas al modelo le toma mas tiempo toma el modelo para computar el aprendizaje.

Subajuste y sobreajuste son otros conceptos para tomar en cuenta al momento de elegir nuestro método de aprendizaje. El subajuste hace referencia cuando el error del set de datos de entrenamiento es grande, mientras que es dice que uno modelos está sobreajustado cuando el error del set de datos de entrenamiento es bajo.En donde el promedio del error es el resultado del uso del método de aprendizaje estadistico para predecir la respuesta de una nueva observación. Usualmente, modelos muy complejos pueden causar que es modelo este sobreajustado.

Al momento de elegir nuestro modelo de aprendizaje debemos de tener en cuenta dos tipos de errores, errores de entrenamiento y errores de test. Los errores de entrenamiento hacen referencia al error de los datos de entrenamiento, mientras que el error de test está asociado con el set observaciones. Algo que hay que tener en cuenta es que para las regresiones lineales y para las clasificaciones, no hay una relación entre el error de entrenamiento y el error de test.
